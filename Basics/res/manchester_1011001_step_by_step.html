Answer — 1 clock cycle per bit, both edges used:
One bit = one full clock period (LOW half + rising edge + HIGH half + falling edge).
Manchester does NOT use 2 full clock cycles. It uses both the rising and falling edge of a single clock cycle as switch points.
That means the wire toggles up to 2× per clock period → wire needs 2× the frequency of NRZ.
What happens at each clock phase (IEEE 802.3 — Bit 1 = L→H, Bit 0 = H→L at midpoint)
— Clock LOW (first half)Signal holds the level set by previous falling edge. Nothing new happens. Receiver waits.
↑ Rising edge (midpoint)Signal FLIPS. This flip IS the bit. L→H = Bit 1. H→L = Bit 0. Receiver latches here.
— Clock HIGH (second half)Signal holds the level it just flipped to. Nothing new happens.
↓ Falling edge (boundary)Setup only. Signal moves to the level needed so the NEXT rising-edge flip encodes the next bit correctly. No data here.
Waveform — data: 1 0 1 1 0 0 1
